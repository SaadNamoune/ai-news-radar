![llm.mojo: Mojo port of llm.c with GPU kernels, multi-GPU ZeRO, FP8/NVFP4 training, and Metal GPU support](https://opengraph.githubassets.com/3f9dcbbc0cdcd552fe23b4b2d1bf51e5f5a0123a2e5bebf27d71120833a89ea6/ulmentflam/llm.mojo)

**Overview:** This is a Mojo port of Andrej Karpathy's llm.c, extending GPU kernels for efficient LLM training and inference. It targets Mojo v1.0.0 and provides an apples-to-apples comparison against llm.c and PyTorch across CPU/GPU/Metal backends. **Method:** The project vendors llm.c as a submodule for CPU/GPU reference and implements fused kernels, multi-GPU ZeRO sharding (stages 0-3), and FP8/NVFP4 quantization for training. ZeRO stages reduce memory via optimizer state, gradient, and parameter sharding with equivalence gating (1e-5 parameter match). FP8 uses delayed scaling for GEMMs, while NVFP4 applies stochastic rounding and Hadamard transforms. **Results:** Benchmarks on NVIDIA DGX Spark show llm.mojo outperforming llm.c and PyTorch in training loop times (e.g., FP8 at 146.6ms vs. 150.5ms pre-optimization). Multi-GPU ZeRO stages reduce memory usage (e.g., stage 1 saves 768MiB fp32) but add collective overhead (negligible at production scales). **Impact:** Advances efficient LLM training on GPUs/Metal, offering a high-performance alternative for researchers and practitioners. Open questions include broader model scale validation and optimization for non-GPU backends.

### [Quantifying Guardrail Gaps in Open-Weight Medical Language Models via Structured Attacks](https://arxiv.org/abs/2607.09804)

<small>**cs.CR updates on arXiv.org** &nbsp;·&nbsp; 2026-07-14 05:00:00 &nbsp;·&nbsp; `medical-ai` `guardrails` `adversarial-attacks` `model-safety`</small>

**Overview:** Evaluates the robustness of MedGemma-4B-it, an open-weight medical LLM, against adversarial attacks targeting prohibited behaviors (e.g., drug dosage recommendations, emergency care advice). Demonstrates that model cards alone are insufficient for enforcing guardrails. **Method:** Constructs a factorial benchmark with 5 guarded behaviors × 50 templated questions × 6 attack types × 3 repetitions (4,500 generations). Uses three independent judges (LLM, regex, NLI-entailment) to classify responses as refuse/hedge/comply. Key attacks include recasting questions as medical board exams (+24.0% ASR) and invoking authority (+14.7% ASR). **Results:** Overall Attack Success Rate (ASR) is 38.0% under the primary LLM judge. Drug-interaction guardrails fail 83.2% of the time, while emergency-deferral guardrails succeed 95.3%. Statistical analysis includes Wilson confidence intervals, cluster-bootstrap effect sizes, and inter-judge reliability (Fleiss' kappa = 0.26). **Impact:** Highlights critical vulnerabilities in open medical models, motivating stronger deployment-time guardrails and robust evaluation frameworks for safety-critical AI.

### [Consensus as Collapse Under Evidence: A Denotational Framework for Distributed Protocols](https://arxiv.org/abs/2607.10152)

<small>**cs.DC updates on arXiv.org** &nbsp;·&nbsp; 2026-07-14 05:00:00 &nbsp;·&nbsp; `distributed-systems` `consensus-protocols` `theoretical-framework`</small>

**Overview:** This paper presents a denotational framework for consensus protocols that reinterprets them as *collapse policies* under evidence derived from communication. It argues that classical consensus outputs (e.g., decided values, replicated logs) are projections of a deeper order-2 evidence state, providing a unified lens to compare protocols. **Method:** The framework models distributed execution as an order-2 evidence state, where consensus protocols specify legitimate evidence, inspection horizons, and collapse policies. It reinterprets impossibility results (e.g., FLP, set agreement) as constraints on evidence accumulation and collapse. The approach abstracts away protocol-specific details to focus on the structural properties of evidence. **Results:** The framework does not introduce new impossibility theorems but provides a formal specification language for consensus protocols, enabling comparisons based on evidence handling. **Impact:** This advances the theoretical foundations of distributed systems by offering a unifying framework to analyze consensus protocols, with implications for designing and verifying fault-tolerant systems.

**Overview:** AfterVibe introduces a framework to extract natural-language specifications from vibe coding sessions (AI-assisted development workflows) by analyzing code artifacts and conversation trajectories. This addresses the challenge of formalizing developer intent in AI-generated code, enabling higher-level review and maintenance. **Method:** The system uses an LLM to generate an abstract specification from the session data, then validates it via a regeneration test: a second LLM re-implements the artifact from the spec alone. A multi-tier validation pipeline grades equivalence between original and regenerated code. Iterative refinement boosts the score to 5.74. The recovered specs outperform human-authored descriptions in guiding regeneration. **Impact:** Demonstrates that specifications—not code—could become the primary artifact for review and record-keeping in AI-driven development, advancing the field of AI-assisted programming and formalizing intent extraction.

### [Impact of Message Format on Multi-Hop LLM Relay Fidelity Across Tiers](https://arxiv.org/abs/2607.09678)

**Overview:** Examines how message format (free NL, JSON, triples, etc.) affects information fidelity in multi-hop LLM relay tasks, addressing conflicting prior work. Introduces a controlled relay testbed with 12 atomic facts re-encoded over 6 hops in 5 formats. **Method:** Tests two relay tiers (strong vs. weak), cognitive-load conditions, and error injection. Evaluates recall accuracy against programmatic ground truth. **Results:** Strong relays are nearly lossless regardless of format (+/-1.8 points), but weak relays show an 8.7x spread in recall (2.3 to 20.5 points). Rigid formats incur encoding tolls, while JSON schemas resist drift. Injected errors persist in 83-100% of chains, matching format retention of true values. **Impact:** Structure enables error-localizing channels but not error correction. Format choice should prioritize the weakest relay in the pipeline, challenging assumptions about format degradation in multi-hop settings.

**Overview:** This work addresses the problem of certifying the diameter of trees in the local certification model, where each node verifies properties based on its local neighborhood. It presents a novel 1-local certification scheme that efficiently certifies whether a tree's diameter is at most *d* using certificates of size at most 3⌈log₂(*d*+1)⌉ bits. **Method:** The scheme leverages the tree's hierarchical structure to encode diameter constraints in logarithmic space, enabling each node to locally verify the global property. The key insight is the use of logarithmic-size certificates to represent the diameter bound, reducing communication overhead in distributed systems. **Results:** The scheme achieves optimal certificate size for tree diameter certification, outperforming general graph approaches that require larger certificates. **Impact:** This advances the field of distributed graph algorithms by providing a theoretically efficient solution for a fundamental problem, with potential applications in network verification and fault-tolerant distributed systems.
