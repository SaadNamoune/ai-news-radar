### [FloatDoor: Platform-Triggered Backdoor Attacks on Generative LLMs via Floating-Point Variability](https://arxiv.org/abs/2606.19535)

<small>**cs.CR updates on arXiv.org** &nbsp;·&nbsp; 2026-06-19 05:00:00 &nbsp;·&nbsp; `LLM-security` `backdoor-attack` `floating-point-arithmetic` `platform-variability`</small>

**Overview:** This paper uncovers a novel attack surface on LLM deployments, *FloatDoor*, a platform-triggered backdoor attack that exploits platform-dependent floating-point arithmetic variability. The attack enables adversary-chosen behavior when the model is served on a target platform while remaining benign otherwise. **Method:** FloatDoor is implemented using two lightweight LoRA adapters: one amplifies inter-platform numerical divergence, and the other binds the platform signature to a malicious task. The attack exploits a time-of-check, time-of-use gap between auditing and serving. **Results:** Demonstrated on Qwen3-4B across NVIDIA GPUs, Google TPUs, AWS Graviton, and Alibaba Yitian-710, FloatDoor reliably induces exploitable code vulnerabilities on target platforms. Aggregate model utility remains largely intact. **Impact:** This work establishes a new class of attacks on LLM deployments, underscoring the urgent need for trusted model supply chains in sensitive applications.

**Overview:** Introduces *computational identifiability*, a finite-sample framework for causal effect estimation that replaces asymptotic theoretical identifiability with an empirical, computation-bound search for estimators within a specified error tolerance. Addresses practical limitations of traditional identifiability assumptions (e.g., infinite data) by framing identifiability as a finite computational procedure. **Method:** Defines a search procedure that outputs an empirical estimator if found within error bounds, conditioned on prior distributions and search assumptions. Leverages probabilistic priors and domain-specific constraints to guide the search. **Results:** Demonstrates efficacy on small samples, ambiguous graphs, mixed observational-interventional data, and counterfactual settings through experiments. Provides open-source code (https://github.com/lbynum/metadentify). **Impact:** Advances causal inference by enabling practical identifiability under finite-sample constraints, opening new avenues for empirical validation in real-world settings where theoretical assumptions fail.

**Overview:** This paper addresses the lack of explicit design decision modeling in model-driven engineering (MDE) transformations, which hinders flexibility, reusability, and traceability. It proposes a formal framework to separate design decisions from transformation implementation for rule-based MDE systems. **Method:** The approach introduces three interconnected models: a decision model (capturing domain-specific design choices), a binding model (mapping decisions to metamodel concepts), and a configuration model (recording chosen options per source element). **Overview:** Subquadratic’s SubQ model claims to solve the quadratic complexity bottleneck of dense attention in transformers by introducing dynamic sparse attention, potentially rendering traditional transformers obsolete. **Method:** SubQ replaces O(n²) dense attention with sparse attention that dynamically selects token pairs based on semantic relevance, avoiding fixed patterns. The architecture leverages sparse matrix operations to reduce multiplications while maintaining performance. **Results:** Third-party evaluation (Appen) reports SubQ 56x faster than FlashAttention in speed tests and 89.7% accuracy on LiveCodeBench (competitive coding), with 12M-token context windows. Cost claims include $8 vs. $2600 for RULER 128 tests, though availability remains limited. **Impact:** If scalable, this could disrupt LLM training/inference economics and energy use. Open questions include generalization to non-coding tasks and robustness across domains.

Scoped semantic memory enables self-learning, while AST-based LLM grading ensures regression-free evaluation. **Results:** The system reduces manual SQL errors by mitigating table discovery ambiguities and simplifies complex queries (e.g., 160-line SQL reduced to agent-driven execution). **Impact:** Advances autonomous data analysis workflows, reducing cognitive load on data scientists and improving query accuracy at scale.
