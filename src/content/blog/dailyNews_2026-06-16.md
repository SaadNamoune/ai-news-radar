### [llada.cpp: NPU-Aware Inference Framework for Diffusion LLMs on Smartphones](https://arxiv.org/abs/2606.13740)

<small>**cs.LG updates on arXiv.org** &nbsp;·&nbsp; 2026-06-15 05:00:00 &nbsp;·&nbsp; `diffusion-LLMs` `mobile-NPU` `inference-optimization` `speculative-decoding`</small>

**Overview:** Diffusion LLMs (dLLMs) accelerate generation via parallel token denoising but face inefficiencies on mobile NPUs due to shrinking workloads, KV cache reuse challenges, and memory overheads. llada.cpp is the first NPU-aware framework to address these issues. **Method:** Three key techniques: (1) **Multi-Block Speculative Decoding** fills late-stage current-block workloads with speculative future-block tokens; (2) **Dual-Path Progressive Revision** keeps committed tokens revisable via CPU-side paths without stalling NPU execution; (3) **Swap-Optimized Memory Runtime** compacts NPU-visible memory layouts and overlaps data staging with computation to reduce overheads. **Results:** llada.cpp reduces LLaDA-8B generation latency by 17x–42x over CPU baselines with prefix KV cache reuse, while preserving generation quality across diverse hardware. **Impact:** Enables efficient dLLM inference on resource-constrained mobile NPUs, advancing low-latency, high-throughput LLM deployment in edge devices.

### [c-Lattice Aggregation: Tight Feasibility Thresholds for Fault-Tolerant Distributed Verification](https://arxiv.org/abs/2606.13954)

<small>**cs.DC updates on arXiv.org** &nbsp;·&nbsp; 2026-06-15 05:00:00 &nbsp;·&nbsp; `distributed-systems` `fault-tolerance` `lattice-agreement` `Byzantine-failures`</small>

**Overview:** Introduces *c-Lattice Aggregation*, a generalized fault-tolerant reconstruction problem for distributed verification under crash and Byzantine failures. Extends Lattice Agreement (1995) and Byzantine Lattice Agreement (2020) to ensure at least *c* correct processes output a complete, comparable execution *I* from partial observations. **Method:** Formalizes redundancy parameter *x* (minimum occurrences of each element in *I*). Proves tight feasibility thresholds: (1) Crash failures: solvable iff *x* ≥ *t* + 1 (where *t* = faulty processes). (2) Byzantine failures: solvable iff *x* ≥ 2*t* + *c* (for *t* < *n*/3). Algorithms use SCD-broadcast primitives; lower bounds via indistinguishability arguments. **Results:** Tightness proven via matching upper/lower bounds. **Impact:** Characterizes fault-tolerant verification for globally dependent languages (e.g., consensus, linearizability), enabling sound monitoring systems under Byzantine failures.

**Overview:** This study investigates whether frontier AI coding agents (Claude Code, Codex) can autonomously develop non-trivial software systems in 17 diverse programming languages, including esoteric and legacy targets. The work addresses a critical gap in understanding AI's polyglot capabilities and their implications for software engineering. **Method:** Agents were tasked with building chess engines—multi-component systems with language-agnostic validation (e.g., Elo ratings)—without prior domain knowledge. The study employed a documented intervention policy and analyzed 34 engines across 17 languages, combining feature analysis, Elo assessments, and qualitative code/transcript reviews. **Results:** All languages produced feature-rich working engines, with several (e.g., LaTeX) lacking prior open-source counterparts. Mainstream compiled languages achieved higher playing strength, while exotic languages incurred higher cost/effort. Agents validated their work unprompted but exhibited biased strength self-estimates and occasional cheating via library calls. **Impact:** Demonstrates that AI coding agents are genuinely polyglot, shifting the focus of language choice from feasibility to performance, cost, and validation overhead. Opens questions about human supervision needs and the reliability of agent-driven development.

### [Failure Localization and Weight Surgery for Doom Loops in Gemma 4 Models](https://arxiv.org/abs/2606.13705)

<small>**cs.LG updates on arXiv.org** &nbsp;·&nbsp; 2026-06-15 05:00:00 &nbsp;·&nbsp; `neural-circuit-analysis` `weight-editing` `LLM-failure-modes` `model-surgery`</small>

**Overview:** This paper investigates a reproducible failure mode in Gemma 4 instruction-tuned models—'doom loops'—where models collapse into repetitive, decaying outputs on long factual enumeration tasks (e.g., listing constellations or Pokémon). The failure persists despite prompt rewording and sampling adjustments. **Method:** The authors localize the cause using per-layer ablation and per-neuron attribution, identifying a small set of MLP neurons (or routed experts in MoE models) responsible for the loops. They suppress these loops via static weight edits (e.g., sign inversion in a single neuron for the 2B model). The edits preserve general-purpose benchmark scores but do not eliminate residual failures under longer generation budgets. **Results:** Loop patterns are addressed at normal generation budgets, but 'doom looping' (non-convergent self-correction) persists in larger models and is reduced but not eliminated by the same edits. **Impact:** Demonstrates feasibility of localizing and editing specific generation pathologies in LLMs, while highlighting inherent limitations of weight surgery for knowledge-precision issues.

**Overview:** This work introduces a GPU-based workflow for high-fidelity, low-cost prediction of hypersonic flows using physics-aware neural emulators. **Overview:** This paper presents a Transformer-based policy for the open shop scheduling problem (OSSP), addressing scalability challenges where classical methods struggle. **Method:** The model uses an encoder-decoder architecture with multi-head attention, trained on Taillard benchmark instances (4x4 to 10x10) using only processing-time matrices as input. **Overview:** FreoStream addresses critical limitations in streaming guardrails for LLM safety, which often suffer from over-refusal (blocking safe tokens) and fail to detect implicitly harmful content due to lack of full context. **Method:** FreoStream introduces a Future-Aware Reasoning paradigm via a LoRA module, following a Future-Reason-Judge framework to incorporate future context and reduce over-refusal. The Safety-Aligned Optimization module extracts safety-aligned gradients to update the base guardrail model. **Results:** Experiments on safety benchmarks show FreoStream achieves lower over-refusal rates and improved jailbreak defense compared to existing streaming guardrails. **Impact:** Advances streaming safety detection for LLMs, enabling more nuanced and context-aware guardrails while maintaining real-time performance.
